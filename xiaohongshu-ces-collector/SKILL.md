# 小红书通用采集筛选器

以“少访问、少输出、不重做”为默认执行原则。CES、时间窗口、粉丝上限和行业词库都只是当次任务的可选参数。

## 核心约束

- 只使用当前请求中的业务规则；未提供的可选规则视为关闭，不继承旧任务。
- 原始正文、评论和页面文本写入任务文件，不直接输出到模型上下文。
- 能由脚本确定的数值、关键词和重复项不交给模型判断。
- 所有核验结果使用 `pass`、`reject`、`unknown`；不得把加载失败当作通过。
- 同一阶段中每篇笔记、每位作者最多访问一次；中断后从检查点继续。
- 只在最终候选和小候补池上执行昂贵的评论审核与语义审核。

## 标准流程

1. 把本次主题、关键词、目标数、候补数、规则、排序和输出字段写入 `work/<run_id>/task.json`。复杂规则或字段不明确时读取 [references/task-contract.md](references/task-contract.md)。
2. 选择一种数据来源并保持到底：用户明确指定 Chrome、Playwright 或现有登录态时服从其选择；不要先试多个浏览器框架。
3. 批量采集搜索卡片并做便宜预筛：明显超期、低互动、禁入标题、重复笔记。
4. 对保留笔记抓取一次详情，核验精确时间、互动数据、正文和作者 ID。详情阶段默认不等待评论。
5. 先执行确定性正文规则，再按 `author_id` 去重核验粉丝。相同作者复用一次结果。
6. 对粉丝通过的紧凑候选包做必要的语义审核；每篇只提供标题、有限正文摘要、命中词和数值字段。
7. 只对目标数加候补数执行评论审核；淘汰后按排序补一个、核验一个，不重跑搜索。
8. 从 `selected.json` 生成交付文件并做数量、公式、链接与打开性检查。

详情页元数据访问与评论审核是两个阶段。评论阶段可能再次打开少量入围笔记；不得声称所有候选“只打开一次”。若详情阶段已经获得可证明完整的评论结果，则直接复用。

## 候选池与停止条件

- 首轮只建立满足目标所需的小候选池。完成一轮后根据实际通过率补量，不固定抓取数百条。
- 卡片不足时扩展关键词；详情、粉丝或评论淘汰导致不足时只补缺口。
- 达到 `目标数 + 候补数` 且全部硬条件已核验后停止。
- 合格数量不足时报告缺口和真实原因，不擅自放宽标准。

## 浏览器与批处理

- 用户指定 Playwright、独立浏览器或不用内置浏览器时，先加载 `playwright`，再读取 [references/playwright.md](references/playwright.md)。
- 用户指定现有 Chrome 标签页时，加载 `chrome:control-chrome` 并复用当前登录态。
- 已授权 API 只有在一次检查即可确认能返回全部必需字段时使用；失败后不循环尝试。
- 任务中不得读取或展示 Cookie、密码、完整浏览器配置或无关页面内容。

## 模型与 Token 预算

- 浏览器脚本返回阶段计数和文件路径，不回传候选数组。
- 禁止把 `raw`、`lines`、整页 `innerText`、完整正文集合或全部评论集合送入模型。
- 语义审核默认每篇正文摘要不超过 350 字；评论只提供命中证据，默认每篇不超过 5 条、每条不超过 160 字。
- 分批审核紧凑候选，写回结构化决定：`note_id`、`decision`、`reason_codes`、`confidence`。
- Excel 技能只在最终导出阶段加载，不在采集阶段读取。

## 状态与恢复

每次任务使用独立目录，至少保留：

```text
work/<run_id>/
  task.json
  state.json
  cards.ndjson
  details.ndjson
  authors.ndjson
  decisions.ndjson
  selected.json
```

每完成一个批次更新 `state.json` 的阶段、成功数、淘汰数、未知数和待处理 ID。验证码或会话中断后只继续待处理项。令牌链接仅保存在任务目录和最终交付文件中，不放进进度消息。

## 按需规则

- Playwright 批处理、粉丝核验和验证码处理：[references/playwright.md](references/playwright.md)
- 评论竞品、博主身份、引流刷屏与完整性：[references/comment-audit.md](references/comment-audit.md)
- Markdown、Excel、公式和可点击链接：[references/output.md](references/output.md)

## 交付沟通

进度最多包含一次阶段性数量更新，以及验证码、登录失效或规则缺失等真实阻塞。最终说明结果数量、未知项、采集时间、评论完整性和文件路径，不展示逐条调试过程。
