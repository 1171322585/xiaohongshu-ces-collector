# 评论审核

只在任务明确要求评论区核验时读取本文件。

## 审核范围

- 只审核通过详情、正文和粉丝硬条件后的目标池与小候补池。
- 浏览器端边加载边判断；普通无风险评论不写入模型审核包。
- 记录已扫描顶层评论数、回复数、`has_more`、停止原因和少量命中证据。
- 连续多次无新增时停止并标记 `partial`，不得冒充完整评论。

## 身份与规则

评论 `user_id == note.author_id` 或带 `is_author` 标签时视为博主。

- `strict_any_competitor`：任何已检查评论或回复命中竞品即淘汰。
- `author_and_spam`：博主本人命中竞品，或评论区存在重复引流刷屏时淘汰；普通用户偶发讨论允许保留。

刷屏需要组合证据，不按单个词机械判断。有效证据包括：同一账号重复推广、相同或高度相似推广文本、多条加微/代订/领券/客服/票务/外链信息。单条普通咨询不构成刷屏。

## 紧凑结果

每篇只保存：

```json
{
  "note_id": "id",
  "status": "pass",
  "coverage": "complete_visible",
  "top_level_checked": 24,
  "replies_checked": 8,
  "reason_codes": [],
  "evidence": []
}
```

`evidence` 默认不超过 5 条，每条截断到 160 字。结论使用“已检查全部可见评论”或“已检查当前可见评论，仍有未加载内容”，不写“永久无竞品”。
