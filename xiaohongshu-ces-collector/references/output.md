# 输出与质量检查

只在生成 Markdown、Excel 或链接清单时读取本文件。

## 数据入口

- 交付器只读取 `selected.json` 和 `task.json`，不重新读取全部候选正文与评论。
- 只生成用户要求的字段；正文默认不进入 Excel。
- 标注采集时间，并说明互动、粉丝和评论是快照。

## Markdown

需要报告时使用 `assets/report-template.md`，删除用户未要求的占位字段。

## Excel

- 仅在此阶段加载 `spreadsheets:Spreadsheets`。
- 用户要求评分列时使用公式并引用规则页权重；同时输出原始互动量时明确区分口径。
- 链接列使用工作簿原生超链接并显示完整 URL。
- 最终 PC 笔记链接保留有效 `xsec_token` 和来源参数；不要用无令牌裸链接替代可用链接。
- 作者主页链接优先使用任务中已验证的完整链接。

导出后检查：数据行数、目标与候补状态、公式数量、公式错误、原生超链接数量、空链接和每张工作表的视觉预览。抽测链接能够返回对应笔记或作者页面。
